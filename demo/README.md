# demo/ — demo assets (not consumed by the build contracts)

Everything here serves the **§21 submission demo** (a ≤5-min recorded video — see
`specs/demo_playbook.md` for demo-mode mechanics, the shot list, seeding, and the
pre-flight checklist).

- **`brand-packs/kanva-coffee/`** — the second demo brand: a FICTIONAL Bengaluru
  small-batch coffee roaster (`ecommerce_dtc` → the product_commerce pack, deliberately a
  different archetype than the AOL brand). `sources/brand-story.md` is what the Strategist
  ingests; `intake-answers.md` is the owner's answer sheet for the recorded interview,
  including the explicitly-elicited fail-closed safety fields. Fictional by design —
  never clone a real brand for a public demo (trademark + publishing risk).
- **`seed/`** *(created during the build, P3+)* — fixtures: 2-week ledger history,
  pre-generated queued pieces, one planted rotation violator, one planted ungrounded
  claim. Shared between tests and the demo.

The first demo brand is the real worked example: `brands/aol/` (educational archetype).
The G1 finale shows both feeds side by side on the same unchanged code.
